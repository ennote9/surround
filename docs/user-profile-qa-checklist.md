# User Profile QA Checklist

Ручная проверка блока **User Profile / Account Center** (Life Progress OS) после этапов **23.1–23.6**. Отмечайте статус при релизе или регрессии.

---

## 1. Scope

**Проверяется**

- Маршрут **`/auth`**: гость (вход/регистрация) и залогиненный пользователь (шлюз).
- Маршрут **`/profile`**: карточки аккаунта, профиля (`display_name`), синхронизации, данных, безопасности.
- Ссылка **«Аккаунт»** в **Sidebar** (desktop).
- Пункт **«Аккаунт»** в меню **«Ещё»** (mobile bottom nav).
- Редактирование **отображаемого имени** и сохранение в **`public.profiles`**.
- **Сброс пароля** (письмо Supabase), **`redirectTo`** = `origin + /auth`.
- **Выход** из аккаунта.
- **Статус синхронизации** (облачное сохранение) на странице профиля.

**Не проверяется в этом чеклисте**

- Удаление аккаунта.
- Смена email в приложении.
- Аватар и загрузка файлов.
- 2FA.
- Тарифы и оплата.
- Полноценная страница смены пароля по recovery session внутри приложения (отдельный будущий этап).

## 12. Known limitations

- Нет удаления аккаунта и смены email в UI.
- Нет аватара, 2FA, тарифов.
- Нет отдельной страницы **«новый пароль»** в приложении после recovery — только письмо Supabase и возврат на **`/auth`**.
- В профиле редактируется только **`display_name`** в **`profiles`**, не «публичный профиль» в соцсетях.

## 14. Final criteria

Блок профиля считается **готовым к релизу**, если одновременно:

1. **`/profile`** стабильно работает на **desktop** и **mobile**.
2. **`display_name`** сохраняется и **переживает** обновление страницы.
3. **Сброс пароля** по email можно **запросить** без ошибок в штатной конфигурации Supabase.
4. **Выход** из аккаунта работает предсказуемо.
5. Навигация **«Аккаунт»** ведёт на **`/profile`**, активные состояния корректны.
6. Нет **заметного** горизонтального overflow на типовых мобильных ширинах.
7. **`npm run build`** и **`npm run lint`** в репозитории проходят.
